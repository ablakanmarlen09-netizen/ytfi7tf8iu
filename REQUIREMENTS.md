### Архитектура нефункционального прототипа

**Тип прототипа:** Нефункциональный (статичные макеты экранов)

**Технологический стек:**
- React 18.3.1 - для создания компонентов-экранов
- React Router 7 - навигация между макетами
- Tailwind CSS 4 - адаптивная стилизация
- TypeScript - типизация данных
- Lucide React - иконки

**⚠️ Отличия от функционального прототипа:**
- ❌ Нет `useState` - состояние не меняется
- ❌ Нет обработчиков событий - кнопки отключены
- ❌ Нет валидации - формы не работают
- ✅ Только статичные данные из `AppContext`
- ✅ Только навигация между экранами (React Router)

**Структура проекта:**
```
src/app/
├── App.tsx              # RouterProvider
├── routes.tsx           # Маршруты для всех 11 экранов
├── contexts/
│   └── AppContext.tsx   # СТАТИЧНЫЕ моки (без методов)
├── components/
│   └── Layout.tsx       # Общий шаблон (header, footer)
└── pages/               # 11 статичных макетов
    ├── Home.tsx         # Главная с баннером
    ├── Catalog.tsx      # Каталог (НФТ аннотация)
    ├── BookDetails.tsx  # Детали книги
    ├── Cart.tsx         # Корзина (статичная)
    ├── Checkout.tsx     # Форма заказа (НФТ аннотация)
    ├── Login.tsx        # Вход (НФТ аннотация)
    ├── Register.tsx     # Регистрация (НФТ аннотация)
    ├── Admin.tsx        # Админ-панель
    ├── AdminAddBook.tsx # Добавление книги
    ├── AdminOrders.tsx  # Управление заказами
    └── NotFound.tsx     # 404
```

### Что осталось в виде описаний (аннотаций)

**Нефункциональные требования, которые невозможно визуально показать:**

| Требование | Почему нельзя показать | Где описано |
|------------|------------------------|-------------|
| **HTTPS шифрование** | Работает на уровне протокола (невидимо пользователю) | Аннотация на `/login`, `/register`, `/checkout` |
| **Хеширование паролей** | Внутренний процесс на сервере (пользователь не видит) | Аннотация на `/login`, `/register` |
| **Время поиска < 2 сек** | Требует реального выполнения функции | Аннотация на `/catalog` |
| **JWT токены** | Работают в фоновом режиме (заголовки HTTP) | Аннотация на `/login` |
| **CSRF защита** | Невидимые токены в формах | Аннотация на `/checkout` |
| **Валидация на сервере** | Серверная логика (не видна в UI) | Аннотация на всех формах |

**✅ Решение:** Все эти требования описаны в **желтых блоках** непосредственно на макетах соответствующих экранов.

### Что потребуется для функционального прототипа

Для перехода от нефункционального к функциональному прототипу нужно добавить:

| Компонент | Текущее состояние | Требуется добавить |
|-----------|-------------------|-------------------|
| **Формы** | disabled (не работают) | Обработчики событий, валидация |
| **Кнопки** | disabled | onClick handlers, изменение состояния |
| **Поиск** | Статичное поле | Фильтрация через useState, useMemo |
| **Корзина** | Статичные товары | addToCart(), removeFromCart(), updateQuantity() |
| **Авторизация** | Макет формы | login(), register(), JWT токены |
| **Админ-функции** | Макеты форм | addBook(), updateOrderStatus() |
| **База данных** | Статичные моки | Supabase / Firebase / API |

## Инструкция по просмотру прототипа

### Навигация по экранам:

1. **Главная** - `/` - баннер, популярные книги, преимущества
2. **Каталог** - `/catalog` - список книг, поиск, фильтры (+ НФТ аннотация)
3. **Детали** - `/book/1` - подробная информация о книге
4. **Корзина** - `/cart` - статичные товары (2 книги)
5. **Оформление** - `/checkout` - форма заказа (+ НФТ аннотация)
6. **Вход** - `/login` - форма входа (+ НФТ аннотация)
7. **Регистрация** - `/register` - форма регистрации (+ НФТ аннотация)
8. **Админ-панель** - `/admin` - статистика
9. **Добавить книгу** - `/admin/add-book` - форма добавления
10. **Заказы** - `/admin/orders` - управление заказами
11. **404** - любой несуществующий URL

### Проверка адаптивности:

1. Откройте DevTools (F12)
2. Toggle Device Toolbar (Ctrl+Shift+M)
3. Выберите различные устройства:
   - iPhone SE (375px) - 1 колонка
   - iPad (768px) - 2 колонки
   - Desktop (1280px) - 3-4 колонки

### Поиск аннотаций НФТ:

Желтые блоки с описанием нефункциональных требований находятся на:
- `/catalog` - производительность поиска
- `/login` - безопасность (HTTPS, bcrypt, JWT)
- `/register` - безопасность (валидация, хеширование)
- `/checkout` - безопасность платежей
- `/admin/add-book` - функциональное требование (добавление книг)
- `/admin/orders` - функциональное требование (управление заказами)
